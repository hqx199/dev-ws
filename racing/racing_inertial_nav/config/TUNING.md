# 避障参数调参指南

## 硬件参考

| 项目 | 尺寸 |
|------|------|
| 小车 | 长30cm × 宽20cm |
| 锥桶 | 底20×20cm × 高30cm |
| 摄像头 | 640×480, 约60° FOV |

## 避障原理

```
障碍物检测 → 在当前位姿前方插入2个临时绕行航点 → Pure Pursuit自动跟踪 → 绕完恢复原始航点

         WP1(侧移)
        / 
       /   锥桶
      /    ▢
  小车*─────────★────── 原始路径 ──────★
                WP2(回正)            原始WP_K
```

WP1 向侧方偏移避开锥桶，WP2 回到原始路径线。Pure Pursuit 全程运行，不需要手动控制角速度/线速度。

---

## 参数详解

### 1. 触发阈值

#### `avoid_area` (默认: 18000)

像素面积阈值。实际生效值为 `avoid_area - 4000`（代码内部偏移）。

- **锥桶 1m 处的面积 ≈ 16000~20000 px²**
- **增大** → 锥桶更近才触发 → 反应更晚
- **减小** → 更远处触发 → 反应更早

| 现象 | 调整 |
|------|------|
| 撞上锥桶（触发太晚） | ↓ 减小到 14000 |
| 绕得太早、太远 | ↑ 增大到 22000 |

> 实际面积受 `calculate_actual_area()` 透视校正影响：图像边缘的锥桶面积会被打折，推迟触发。

#### `avoid_trigger_bottom` (默认: 280)

锥桶底部在图像中的 y 坐标阈值 (0~480)。

- **值越大** → 锥桶必须离车更近（画面更靠下）才触发
- **值越小** → 更早触发

| 现象 | 调整 |
|------|------|
| 来不及绕 | ↓ 减小到 240 |
| 太远就触发 | ↑ 增大到 320 |

---

### 2. 绕行几何

#### `avoid_detour_lateral_offset` (默认: 0.4m)

WP1 的侧向偏移距离。

```
侧移 40cm = 半车身(10cm) + 半锥桶(10cm) + 安全余量(20cm)
Clearance ≈ 20cm（车身外沿到锥桶外沿的最小距离）
```

| 现象 | 调整 |
|------|------|
| 擦边撞上 | ↑ 增大到 0.5 |
| 绕太远（偏离路线） | ↓ 减小到 0.3 |

> 注意：Pure Pursuit 的 min_turning_radius=0.3m 将最大曲率限制在 3.33 rad/m。侧移 0.4m + lookahead 0.5m → 曲率 3.2 rad/m，刚好可达。

#### `avoid_detour_forward_step` (默认: 0.6m)

每个绕行航点的前进距离。WP1 在前方 0.6m，WP2 在前方 1.2m。

- **必须 > PP lookahead_distance(0.5m)** 才能形成平滑曲线
- **太小** → PP 转弯过急，可能超出最小转弯半径
- **太大** → 车直行太久才开始绕，可能太晚

| 现象 | 调整 |
|------|------|
| 急转弯擦边 | ↑ 增大到 0.8 |
| 绕得太远 | ↓ 减小到 0.5 (需同时调整 lookahead) |

---

### 3. 冷却

#### `avoid_detour_cooldown` (默认: 3.0s)

两次绕行之间的最小间隔。绕行完成（到达 WP2）后开始计时。

- **防止反复触发**：绕行完成瞬间锥桶可能仍在视野中，冷却避免立马再绕
- 车以 0.4 m/s 前进，3 秒内走 1.2m → 足够远离锥桶

| 现象 | 调整 |
|------|------|
| 反复触发同一个锥桶 | ↑ 增大到 5.0 |
| 碰上第二个锥桶不反应 | ↓ 减小到 1.5 |

---

### 4. 速度

#### `avoid_linear_speed` (默认: 0.4 m/s)

绕行期间的线速度上限（通过临时降低 max_linear_speed 实现）。

| 现象 | 调整 |
|------|------|
| 绕行时转向来不及 | ↓ 减小到 0.3 |
| 绕行太慢 | ↑ 增大到 0.5 |

---

## 快速调参口诀

```
撞上了         → area↓ bottom↓ lateral↑ forward↑ speed↓
绕太早/太远    → area↑ bottom↑ lateral↓ forward↓
转弯擦边(太急) → forward↑ speed↓
转弯不够猛     → lateral↑ forward↓ (增大曲率)
反复绕同一个   → cooldown↑ speed↑
```

## 调试方法

```bash
# 查看绕行触发日志
grep "DETOUR\|DETOUR complete" /tmp/nav*.log

# 实时查看当前位置和航点
ros2 topic echo /racing_obstacle_detection --no-arr --once --field targets

# 查看 cmd_vel 确认 PP 在正常输出
ros2 topic echo /cmd_vel --once
```

## 典型调参流程

1. **先确认触发正常**：放锥桶，看日志是否输出 "DETOUR!"
2. **调触发距离**：太近撞上 → `avoid_area↓` / `bottom↓`；太远 → `avoid_area↑` / `bottom↑`
3. **调绕行轨迹**：擦边 → `lateral↑`；转弯太急 → `forward↑`
4. **调冷却**：反复触发 → `cooldown↑`

> 修改 `params.yaml` 后无需重新编译，重启节点即可生效：
> ```bash
> bash run_inertial_nav.sh
> ```
