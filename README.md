面向医疗赛事的 RDK X5 OriginCar 智能配送小车技术报告
一、标准化竞赛三步流程
1.扫码取任务阶段：小车从病房发车点 P 自动驶出，自主行驶至二维码任务点位，解析二维码获取药品配送 / 病房巡诊任务与通道绕行方向；
2.通道人形识别阶段：按照二维码指定顺 / 逆时针沿医疗通道行驶，持续检测通道内医护人形立牌，筛选清晰图像调用大模型生成图文描述，同步语音播报识别结果；
3.折返停车收尾阶段：完成全通道巡诊任务后自主折返发车点 P，识别停车标识完成精准入库，语音播报任务完成，整套竞赛流程结束。
二、五大核心功能模块
1.二维码识别模块 基于 pyzbar 库实现二维码解码，兼容深度相机、单目 USB 相机双路图像输入；解析二维码存储的医疗任务、通道绕行指令，将任务信息发布至/qr_detector/task_info话题，总控节点接收后切换病房通道行驶路线。
2.人形检测与大模型交互模块 采用 YOLOv8 算法识别人形医护立牌，搭配深度相机过滤远距离无效干扰；多帧图像择优筛选清晰画面，支持本地轻量化 VLM、云端 GPT-4o 两种大模型切换；对人形图像进行图生文语义解析，输出医护标识信息发布至/llm_interface/image_caption话题。
3.导航避障模块 融合 IMU + 轮式编码器 EKF 多传感器定位，搭载单线 TOF 激光雷达 + YOLO 视觉双重防撞机制；支持病房通道顺时针、逆时针双路径规划，内置病房电子活动围栏；可自定义安全距离、行驶车速、航点定位容差、弯道减速系数，适配狭窄医疗通道场景。
4.语音播报模块 订阅二维码任务话题、人形识别大模型输出话题、车辆导航状态话题；实时播报巡诊任务类型、医护人员识别信息、车辆限速提醒、到达病房点位、停车完成等医疗场景语音提示，提升竞赛交互效果。
5.任务管理主节点 系统顶层调度核心，采用有限状态机管控完整医疗竞赛流程；统一管理任务计时、多阶段自动切换、感知数据优先级调度；识别超时、场地越界等异常场景自动执行减速、回退应急逻辑，统筹所有感知、导航、语音模块协同运行。
三、硬件配套需求
主控设备：地平线 RDK X5 开发板，向下兼容 RDK X3、S100 系列；车体平台：OriginCar 阿克曼转向机器人底盘； 传感器套件：单线 TOF 激光雷达、Aurora930 深度 RGB 相机、USB 单目相机、BMI088 六轴 IMU、500 线增量轮式编码器； 外设配件：语音播报扬声器、STM32 底层运动控制板。
四、项目工程结构
标准 ROS2 工作空间ros2_ws，2026 年 6 月正式发布，开源协议 Apache 2.0；src 源码目录划分 5 个独立功能包，配套一键部署脚本、参数配置文件、状态监控工具：
1.qr_detector：医疗任务二维码识别包
2.human_detector：人形医护标识检测 + 本地 VLM 图生文包
3.navigation：多传感融合导航 + 激光视觉双避障包
4.voice_output：医疗场景语音播报功能包
5.task_manager：医疗竞赛全流程总控调度包
五、启动方式
1.一键启动完整智慧医疗竞赛系统：ros2 launch task_manager competition.launch.py
2.分模块独立调试：各功能包配套专属 launch 启动文件，可单独启动二维码识别、人形检测、导航、语音、总控节点，离线调试各模块功能。
六、关键可配置参数
1.导航避障参数：激光雷达安全距离 0.5m，整车最大行驶速度 0.5m/s，病房航点定位容差 0.3m，弯道限速系数 0.6；
2.大模型交互参数：本地 VLM 开关、本地模型存储路径、云端大模型 API 密钥、图像采样间隔 3 秒；
3.视觉感知参数：YOLO 人形检测置信度 0.4，二维码识别超时阈值 10 秒，语音播报音量、播报延迟时长；
4.安全约束参数：病房电子围栏边界坐标、车辆越界回退速度、单阶段任务最长计时阈值。
七、环境部署
项目提供install_dependencies.sh自动化部署脚本，一键完成全项目依赖安装，包含 ROS2 机器人运行环境、地平线 BPU 推理 SDK、YOLO 目标检测框架、本地 VLM 视觉大模型、语音合成库、多传感器融合定位组件；脚本自动配置串口、I2C、相机硬件访问权限，部署完成无需额外底层配置。
